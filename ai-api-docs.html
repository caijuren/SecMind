SecMind API 参考

SecMind 是一款 AI 驱动的安全运营平台 (AI Security Operations Platform)，提供 告警管理、AI 智能分析、威胁狩猎、自动化剧本响应、多租户管理等核心能力。 所有 API 请求均通过 HTTPS 发送至 https://api.secmind.com/api/v1。

平台基于 FastAPI 构建，采用 JWT Bearer Token 鉴权，支持 RBAC 细粒度权限控制。以下文档覆盖所有公开 API 端点。

Base URL
https://api.secmind.com/api/v1
认证方式
Bearer Token (JWT)
版本
v1

# 快速开始

使用 API 前，首先需要注册账户并获取 API 密钥。所有受保护的端点要求在请求头中携带 Authorization: Bearer <token>。

开发者可通过 GET /api/v1/health 检测服务状态，或访问 GET /api/v1/api 查看 Scalar 交互式 API 文档。

# 认证

平台使用 JWT (HS256) 进行鉴权。访问令牌有效期 60 分钟，过期后可使用刷新令牌获取新的访问令牌。 连续登录失败 5 次将触发账户锁定。

POST /auth/login 使用邮箱和密码登录，返回访问令牌和刷新令牌
参数类型描述
email string required 用户注册邮箱
password string required 用户密码

频率限制: 5 次/分钟 · 响应: Token (access_token, refresh_token, token_type)

POST /auth/register 注册新用户，自动返回令牌
参数类型描述
email string required 用户邮箱
password string required 密码
name string optional 用户姓名
phone string optional 手机号

频率限制: 3 次/分钟 · 响应: RegisterResponse

POST /auth/refresh 使用刷新令牌获取新的访问令牌
参数类型描述
refresh_token string required 登录时获得的刷新令牌
POST /auth/send-sms-code 发送短信验证码
参数类型描述
phone string required 手机号

频率限制: 3 次/分钟 · 验证码发送后可使用 POST /auth/phone-login 登录

GET /auth/captcha 获取数学验证码图片

返回一张包含数学题目的图片。用户需通过 POST /auth/verify-captcha 提交答案进行验证。

GET /auth/me 获取当前登录用户信息

需要 Bearer 令牌。返回 UserRead，包含 id、name、email、phone、department、position、avatar_url 等字段。

# 告警 & AI 分析

告警管理是 SecMind 的核心能力。平台支持多维度过滤告警、AI 自动分析研判、 风险评分与处置建议。

GET /alerts 获取告警列表，支持多维度筛选

查询参数

参数类型默认描述
type string optional null 告警类型过滤
risk_level string optional null 风险等级过滤（critical / high / medium / low）
status string optional null 状态过滤（open / investigating / resolved / closed）
search string optional null 搜索关键词（匹配标题和描述）
skip integer optional 0 分页偏移量
limit integer optional 20 每页数量

响应: AlertListResponse — 包含 items (AlertRead[])、total、skip、limit

GET /alerts/{alert_id} 获取单条告警详情

返回完整的告警对象，包含 type、title、description、risk_level、status、 source_ip、destination_ip、timestamp、ai_score、ai_summary、ai_recommendation 等字段。

PUT /alerts/{alert_id}/status 更新告警处理状态
参数类型描述
status string required 新状态：open / investigating / resolved / closed

AI 智能分析

AI 分析模块对告警进行自动化研判，支持单条分析和批量提交。分析结果包含 威胁评分、处置建议、证据链等信息。

GET /ai-analysis/events 获取分析事件列表
参数类型描述
limit integer optional 返回数量，默认 20
severity string optional 严重程度过滤
status string optional 分析状态过滤
source string optional 数据来源过滤
POST /ai-analysis/events 提交告警进行 AI 分析
参数类型描述
alert_id integer required 要分析的告警 ID

也支持批量提交：POST /ai-analysis/events/batch，请求体为 BatchAnalyzeRequest

GET /ai-analysis/stats 获取 AI 分析统计概览

返回分析总量、各严重程度分布、平均响应时间等统计数据。

# AI 对话

AI 对话模块提供安全助手功能：通过与 LLM 交互进行告警调查分析、 生成研判报告、查询 IOC 信息等。所有对话记录持久化保存。

GET /ai-chat/sessions 获取当前用户的对话会话列表

需要 Bearer 令牌。返回按更新时间倒序排列的会话列表，每个会话包含 id、title、context、alert_id。

POST /ai-chat/sessions 创建新的对话会话
参数类型描述
title string required 会话标题
alert_id integer optional 关联的告警 ID（可选）
POST /ai-chat/sessions/{session_id}/messages 在指定会话中发送消息
参数类型描述
content string required 用户输入的消息内容
tool_calls object optional 触发 AI 工具的调用配置

AI 返回的消息包含 role、content、tool_calls、tool_results、confidence 等字段。 可用的 AI 工具列表通过 GET /ai-chat/tools 获取。

POST /ai-chat/sessions/{session_id}/report 基于会话内容生成安全调查报告

自动将对话内容整理为结构化报告。完整的报告管理还包括： GET /ai-chat/reports（报告列表）、 POST /ai-chat/reports（创建报告）、 PUT /ai-chat/reports/{report_id}（更新报告）。

# 安全运营

安全运营模块覆盖响应动作执行、威胁狩猎、自动化剧本编排（DAG 引擎） 等核心 SOAR 能力。

GET /response/actions 获取响应动作列表，支持过滤
参数类型描述
status string optional 过滤：pending / approved / executing / completed / failed / cancelled
action_type string optional 动作类型过滤
priority string optional 优先级过滤

响应: 每个动作包含 action_type、target、status、priority、alert_id、ai_reasoning、reasoning_chain、evidence_summary、guardrails 等字段

POST /response/actions/{action_id}/execute 执行指定的响应动作
POST /response/actions/{action_id}/approve 审批响应动作

威胁狩猎

GET /hunting/hypotheses 获取威胁狩猎假设列表
参数类型描述
status string optional 过滤：draft / active / validated / invalidated
tactic string optional MITRE ATT&CK 战术编号
confidence number optional 最低置信度阈值

自动化剧本 (DAG 引擎)

剧本编辑器支持可视化编排安全响应流程，基于 DAG（有向无环图）引擎执行。 每个剧本由 nodes（节点）和 edges（连线）定义。

GET /playbooks 获取剧本列表
POST /dag/validate 验证 DAG 定义（节点和边）的正确性
POST /dag/execute/{playbook_id} 执行指定剧本的 DAG 流程

执行状态可通过 GET /dag/status/{execution_id} 实时查询。

GET /playbook-editor/templates 获取节点模板列表，按分类组织

# 系统管理

系统管理模块提供用户、角色权限（RBAC）、多租户、计费、合规等管理能力。 需要对应权限（如 users:read、rbac:write）。

GET /users 获取用户列表（需要 users:read 权限）
POST /users 创建新用户（需要 users:write 权限）
参数类型描述
name string required 用户名
email string required 邮箱
password string required 密码
department string optional 所属部门
position string optional 职位

RBAC 权限管理

GET /rbac/roles 获取所有角色列表
GET /rbac/permissions 获取所有权限定义
PUT /rbac/users/{user_id}/roles 为用户分配角色
参数类型描述
role_ids array[integer] required 要分配的角色 ID 列表

支持的资源权限前缀示例：alerts:read、alerts:write、 response:execute、playbooks:write、 tenants:read、billing:read 等 20+ 个资源类别。

多租户管理

POST /tenants 创建新租户
参数类型描述
name string required 租户名称
slug string required 唯一标识符
plan string optional 套餐：starter / pro / enterprise
owner_email string required 租户管理员邮箱
GET /tenants/{tenant_id}/quota 获取租户配额使用情况
POST /tenants/{tenant_id}/members 添加租户成员
参数类型描述
user_id integer required 用户 ID
role string required 成员角色：admin / member / viewer
GET /system-settings 读取系统全局设置
GET /system/perf 获取系统性能统计

缓存周期 30 秒。返回 CPU 使用率、内存使用量、请求计数、平均响应时间等指标。

# 情报 & 集成

IOC 查询模块集成 VirusTotal、AbuseIPDB、ThreatFox 等外部情报源， 支持 IP、域名、Hash、URL 等多种 IOC 类型的自动识别与批量查询。

GET /ioc/lookup/{ioc_value} 查询 IOC 威胁情报
参数类型描述
ioc_value string required IOC 值（IP / 域名 / Hash / URL），系统自动识别类型
ioc_type string optional 手动指定 IOC 类型：ip / domain / hash / url
POST /ioc/batch 批量 IOC 查询
参数类型描述
iocs array[string] required 最多 100 个 IOC 值

集成管理

GET /integrations/apps 获取已配置的集成应用列表
GET /integrations/adapters 获取集成适配器列表
POST /integrations/webhooks/inbound 接收外部系统入站 Webhook

# 错误处理

API 返回标准 HTTP 状态码和 JSON 错误体。每个错误包含 detail 字段（中文描述）， 部分错误还包含 param 指明导致错误的参数字段。

状态码说明常见原因
200 请求成功 正常返回数据
400 请求参数错误 缺少必填参数、参数格式错误、数据已存在
401 未授权 缺少或无效的 Bearer Token、令牌过期
403 权限不足 当前角色缺少对应资源的操作权限
404 资源不存在 请求的资源 ID 不存在
429 请求频率超限 超过速率限制，检查 Retry-After 响应头
500 服务器内部错误 服务器内部错误，请稍后重试

错误响应示例

{ "detail": "邮箱或密码错误" }

重试策略

针对 429 和 5xx 错误，建议使用指数退避 + 随机抖动（jitter）：

  • 第一次重试：等待 1 秒
  • 第二次重试：等待 2 秒
  • 第三次重试：等待 4 秒
  • 第四次重试：等待 8 秒
  • 最多重试 5 次，之后优雅降级

对于 429 响应，Retry-After 响应头优先级高于默认退避策略。

代码示例

login.py 
import requests

BASE_URL = "https://api.secmind.com/api/v1"

# Step 1: 登录获取令牌
resp = requests.post(f"{BASE_URL}/auth/login",
    json={
        "email": "admin@example.com",
        "password": "your-password"
    })
token = resp.json()["access_token"]
headers = {"Authorization": f"Bearer {token}"}

# Step 2: 获取告警列表
alerts = requests.get(f"{BASE_URL}/alerts",
    params={"risk_level": "high", "limit": 10},
    headers=headers).json()
print(f"获取到 {alerts['total']} 条高危告警")
ai_analysis.py 
import requests

BASE = "https://api.secmind.com/api/v1"
headers = {"Authorization": "Bearer YOUR_TOKEN"}

# 提交告警进行 AI 分析
analysis = requests.post(f"{BASE}/ai-analysis/events",
    json={"alert_id": 123}, headers=headers).json()

# 查询分析结果
result = requests.get(
    f"{BASE}/ai-analysis/events/{analysis['id']}",
    headers=headers).json()

print("威胁评分:", result.get("ai_score"))
print("处置建议:", result.get("ai_recommendation"))

login.js 
const BASE_URL = "https://api.secmind.com/api/v1";

// 登录获取令牌
const loginResp = await fetch(`${BASE_URL}/auth/login`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    email: "admin@example.com",
    password: "your-password"
  })
});
const { access_token } = await loginResp.json();

// 获取高危告警
const alertsResp = await fetch(
  `${BASE_URL}/alerts?risk_level=high&limit=10`,
  { headers: { "Authorization": `Bearer ${access_token}` } }
);
const alerts = await alertsResp.json();
console.log(`获取到 ${alerts.total} 条高危告警`);
