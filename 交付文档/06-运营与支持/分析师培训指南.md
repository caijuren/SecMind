# SecMind AI安全运营平台 — 分析师培训指南

**版本**：v2.3.0  
**适用对象**：安全分析师、安全运营人员  
**预计培训时长**：3天（理论1.5天 + 实操1.5天）

---

## 培训目标与前提条件

### 培训目标

完成本培训后，学员将能够：

1. 熟练使用SecMind平台进行日常安全运营工作
2. 掌握AI辅助告警研判的操作方法，正确理解AI推理结果
3. 能够使用AI自主调查功能展开深度威胁分析
4. 掌握案件全生命周期管理和研判闭环流程
5. 熟练使用自动化处置功能快速响应安全威胁
6. 掌握威胁狩猎的基本方法和技术
7. 能够创建和发布安全响应剧本
8. 掌握合规报表的查看、导出和自定义配置

### 前提条件

学员应具备以下基础知识和技能：

- **安全基础知识**：了解常见的网络攻击类型（如勒索病毒、钓鱼攻击、DDoS、暴力破解等）
- **安全工具经验**：使用过SIEM、EDR等安全工具的基本操作
- **告警分析经验**：具备基本的告警分析和威胁研判经验
- **MITRE ATT&CK框架**：了解MITRE ATT&CK攻击框架的基本概念

### 培训环境

- 已部署SecMind v2.3.0平台（已部署并可访问）
- 具有分析师角色权限的培训账号
- 培训数据集：内置模拟告警数据集（包含真实攻击场景模拟数据）

## 模块2：告警分析与AI辅助研判

### 2.1 告警详情查看

点击告警列表中的任意告警，进入告警详情页，包含以下信息：

**基本信息：**
- 告警ID、名称、原始告警日志、源IP/端口、目标IP/端口
- 协议、告警类型、严重级别、发生时间、采集时间
- 数据源名称、原始告警ID

**AI研判结果区（核心）：**
- AI判定结果：真实告警（True Positive）/ 误报（False Positive）/ 存疑（Suspicious）
- 置信度分数：0-100的数字展示，附带进度条和颜色标识
  - 90-100：极高置信度（绿色）
  - 80-89：高置信度（深绿）
  - 60-79：中等置信度（黄色）
  - <60：低置信度（红色）
- AI推理依据：模型给出判断的具体原因分析
- 建议处置方案：AI推荐的处置动作和剧本
- 关联MITRE ATT&CK：攻击技术编号和阶段
- 威胁情报匹配结果：关联的IOC情报信息

**上下文信息（Tab页）：**
- 原始日志：原始告警日志原文
- 相似告警：同一源/目标的历史告警记录
- 关联告警：可能属于同一攻击链的其他告警
- 资产信息：受影响资产的详细信息（如已对接CMDB）
- 关联案件：该告警已关联的案件信息

### 2.2 理解AI推理结果

**AI研判结果解读示例：**

```
┌─────────────────────────────────────────────────┐
│  AI研判结果                                       │
│                                                  │
│  判定：真实告警 (True Positive)                    │
│  置信度：92/100  ████████████░░  极高置信度        │
│                                                  │
│  推理依据：                                       │
│  1. 源IP 10.0.1.100 在过去1小时内曾对5台内网服务器 │
│     发起SSH暴力破解，共计1,234次失败登录尝试         │
│  2. 目标服务器 172.16.0.50 于10:23:15成功登录      │
│     非工作时间、非授权管理员操作                     │
│  3. 成功登录后30秒内触发了异常外连行为               │
│  4. 该行为模式与已知的"X-Crypter勒索病毒"传播行为    │
│     高度相似（相似度87%）                           │
│                                                  │
│  MITRE ATT&CK映射：                               │
│  - T1110 (暴力破解：Brute Force)                   │
│  - T1078 (有效账号：Valid Accounts)               │
│  - T1059 (命令执行：Command and Scripting)        │
│                                                  │
│  建议处置方案：                                    │
│  ⚠ 立即隔离源IP 10.0.1.100                       │
```

**关键说明：**

- AI判定为**真实告警**高置信度时：一般情况下可直接信任，按照建议处置方案操作
- AI判定为**误报**高置信度时：可直接驳回，系统会自动学习该模式
- AI判定为**存疑**或**低置信度**时：需人工核查，此时系统会自动标记为"待人工复核"
- AI推理依据可展开查看详细分析过程，帮助分析师理解AI的判断逻辑

### 2.3 确认与驳回操作

**确认告警（标记为真实告警）：**

1. 在告警详情页点击「确认真实告警」按钮
2. 可选操作：关联到现有案件 / 创建新案件 / 仅确认不关联
3. 可添加分析备注（非必填，但建议填写）
4. 点击「确认」，系统记录分析师确认信息反馈至AI模型

**驳回告警（标记为误报）：**

1. 在告警详情页点击「驳回」按钮
2. 选择驳回原因：
   - 已知误报（已配置白名单规则）
   - 测试流量
   - 误判（请说明原因）
   - 重复告警
   - 其他（自定义填写）
3. 可选添加备注说明
4. 点击「驳回」，告警状态更新为"已驳回"
5. 系统将驳回原因反馈给AI模型，用于后续改进

**批量操作：**

1. 在告警列表中勾选多条告警（支持全选）
2. 点击顶部「批量确认」或「批量驳回」
3. 确认操作弹窗，添加统一备注
4. 执行确认或驳回

### 2.4 告警过滤与搜索技巧

**快速过滤：**

- 点击严重级别标签：快速筛选对应级别的告警
- 点击AI状态：快速筛选未分析/待复核告警
- 点击IP地址：查看同一IP的所有告警活动

**高级搜索：**

支持以下搜索语法：

**保存搜索条件：**

1. 设置好过滤条件后，点击「保存搜索」
2. 输入保存名称（如"每日待复核告警"）
3. 保存后可在侧边栏快速过滤下拉框中直接选择

## 模块3：AI自主调查使用指南

### 3.1 什么是AI自主调查

AI自主调查是SecMind的核心差异化功能。当一条告警被确认为真实威胁后，分析师可以触发AI Agent自动展开深度调查。AI会像一个经验丰富的安全分析师一样：

1. 收集相关上下文信息（关联告警、资产信息、威胁情报等）
2. 逐层深入挖掘攻击链路
3. 分析攻击手法和影响范围
4. 生成完整的调查推理报告
5. 提供处置建议和证据链

### 3.2 触发AI调查

**手动触发：**

1. 在告警详情页点击「AI自主调查」按钮
2. 系统弹出配置弹窗：
   - 调查深度：快速（5-10分钟）/ 标准（15-30分钟）/ 深度（30-60分钟）
   - 调查范围：仅当前告警 / 关联告警 / 全量关联信息
   - 额外情报：是否启用外部威胁情报查询
3. 点击「开始调查」

**自动触发（需管理员配置）：**

- 当AI判定置信度>95的真实告警，系统默认自动触发AI调查
- 自动调查默认使用"快速"模式

### 3.3 查看调查过程

**调查进度展示：**

AI调查开始后，系统以实时更新的方式展示调查进度：

```

│  ▷ 分析攻击链路                 ████░░░░░░ 40%    │
```

**3. 受影响资产清单**

### 3.5 复核调查结论

**操作流程：**

1. 仔细阅读AI调查报告，逐项核实关键发现
2. 点击「查看证据」可展开查看每项证据的详细内容和来源
3. 对于AI报告中的每一项发现，分析师可以：
   - 同意（Agree）：标记为已确认
   - 部分同意（Partial）：修改后确认
   - 不同意（Disagree）：给出理由并进行修正
4. 完成复核后，点击「确认调查结论」
5. 选择后续操作：创建案件 / 关联到现有案件 / 触发自动处置

**复核要点：**

- 时间线是否逻辑自洽
- 证据链是否完整
- 攻击链路分析是否合理
- 受影响资产范围是否全面
- 处置建议是否适当

## 模块4：案件管理与研判闭环

### 4.1 创建案件

**手动创建：**

1. 进入「案件管理」→「新建案件」
2. 填写案件信息：
   - 案件名称：简明扼要描述案件内容，如"XX银行服务器遭勒索病毒攻击"
   - 严重级别：根据实际威胁程度选择
   - 案件类型：恶意软件/钓鱼攻击/数据泄露/权限滥用/其他
   - 关联告警：选择需要关联到该案件的告警
   - 案件描述：详细描述案件的来龙去脉
   - 标签：自定义标签，便于后续检索
3. 保存创建

**从告警创建：**

在告警详情页点击「创建案件」，系统自动将该告警作为案件的首条证据。

**AI自动创建：**

当AI调查发现确凿攻击证据时，系统可自动生成案件草稿，分析师审核确认后正式创建。

### 4.2 案件工作台

案件详情页包含以下Tab：

| Tab | 内容 |
|-----|------|
| 概览 | 案件基本信息、状态、处理人、时间线等 |
| 告警列表 | 关联到该案件的所有告警 |
| 证据库 | 案件的证据文件、截图、日志等 |
| 调查记录 | AI调查报告、人工调查记录、分析笔记 |
| 处置记录 | 已执行的处置动作和结果 |
| 协作评论 | 团队成员之间的讨论记录 |
| 时间线 | 案件全生命周期的时间轴视图 |

### 4.3 添加证据

在「证据库」Tab中：

**添加证据：**

1. 点击「添加证据」
2. 选择证据类型：日志文件 / 截图 / PCAP包 / 文件样本 / 邮件 / 其他
3. 上传文件（支持拖拽上传，单文件最大100MB）
4. 填写证据描述
5. 如需标记关键证据，勾选「标记为关键证据」

**证据管理：**

- 支持证据标签分类（网络证据、主机证据、用户证据等）
- 支持证据版本管理（同一证据可上传更新版本）
- 关键证据自动置顶显示
- 证据完整性校验（SHA256）

### 4.4 关联告警

**手动关联：**

1. 在案件详情页点击「关联告警」
2. 搜索需要关联的告警（支持ID搜索和条件筛选）
3. 勾选目标告警，点击「确认关联」

**自动关联建议：**

系统基于AI分析，会自动推荐与该案件相关的其他告警：
- 案件详情页中「推荐关联」区域显示待关联告警列表
- 每条推荐附带关联理由和相似度评分
- 分析师一键确认关联或手动调整

### 4.5 案件流转状态

```
待处理 ─→ 调查中 ─→ 待处置 ─→ 已结案
  │          │         │
  └──────────┴─────────┘→（任何状态可退回待处理）
```

**状态变更操作：**

| 操作 | 说明 | 触发条件 |
|------|------|---------|
| 开始调查 | 将案件状态从"待处理"变更为"调查中" | 分析师接手案件 |
| 请求处置 | 调查完成后将案件变更为"待处置" | 需要执行处置动作 |
| 结案 | 完成全部处置后，将案件变更为"已结案" | 确认威胁已清除 |
| 退回 | 将案件退回"待处理"状态 | 需要重新评估或重新分配 |

**结案要求：**

结案前必须完成以下操作：
1. 所有关联告警已处理（确认/驳回）
2. 已执行的处置动作记录完整
3. 案件调查报告已填写
4. 如适用，已提交合规报告

### 4.6 协作功能

**案件评论：**

- 在「协作评论」Tab中可发表评论
- 支持@提及团队成员（被提及人将收到通知）
- 支持回复和评论置顶

**案件分配：**

- 案件详情页顶部的"处理人"字段支持变更
- 可分配给指定分析师或分析团队
- 分配后系统自动通知接手的分析师

**案件导出：**

- 支持导出案件摘要（PDF格式）
- 支持导出案件证据包（ZIP格式）
- 支持导出完整案件报告（含所有告警、调查记录、处置记录）

## 模块5：自动化处置操作

### 5.1 一键处置

**操作路径：**

1. 在告警详情页或案件详情页点击「处置」按钮
2. 系统显示AI推荐的处置方案列表：
   ```
   推荐处置方案（AI评分：92）：
   ┌─────────────────────────────────────────┐
   │ ☑ 在防火墙上封禁源IP 10.0.1.100        │
   │   目标设备：FortiGate-02                 │
   │   操作：添加黑名单规则，有效期24小时       │
   ├─────────────────────────────────────────┤
   │ ☑ 锁定被盗账号 admin_zhang              │
   │   目标系统：AD域控                       │
   │   操作：禁用账号，强制下线                 │
   ├─────────────────────────────────────────┤
   │ ☐ 隔离受影响服务器 172.16.0.50          │

   └─────────────────────────────────────────┘
   ```

3. 勾选需要执行的处置方案
4. 点击「执行处置」，系统弹出确认弹窗
5. 确认后，系统开始逐项执行处置动作
6. 处置完成后展示结果：
   ```
   处置结果：
   ✓ 防火墙规则已添加（规则ID：ACL-20260518-001）
   ✓ 账号已禁用（操作耗时：2.3秒）
   ✗ 服务器隔离失败：NSX API超时，请手动操作或稍后重试
   ```

### 5.2 批量处置

**操作路径：**

1. 在告警列表中勾选多条同类型的告警
2. 点击顶部「批量处置」
3. 选择统一的处置动作：
   - 批量封禁源IP（合并同一IP）
   - 批量隔离主机
   - 批量锁定账号
   - 批量告警确认
4. 配置处置参数
5. 确认执行

**批量处置的智能处理：**

- 系统自动去重：多条告警指向同一源的，自动合并为一次处置
- 冲突检测：检测即将执行的处置是否与现有策略冲突
- 风险评估：评估批量操作的风险等级，高风险操作需二次确认

### 5.3 处置记录查看

进入「自动化处置」→「处置记录」，可查看：

| 字段 | 说明 |
|------|------|
| 处置ID | 唯一标识 |
| 关联告警/案件 | 触发处置的告警或案件ID |
| 处置动作 | 执行的具体操作 |
| 目标设备 | 执行处置的目标系统 |
| 状态 | 执行中/成功/失败/部分成功 |
| 操作人 | 手动处置的操作人 / 系统自动 |
| 执行时间 | 处置动作的执行时间 |
| 结果详情 | 详细的执行结果和返回信息 |

### 5.4 回滚操作

**回滚条件：**

- 处置后发现是误判
- 处置导致业务异常
- 需要恢复被阻断的合法流量

**回滚操作：**

1. 在「处置记录」中找到需要回滚的处置
2. 点击「回滚」按钮
3. 系统自动执行反向操作：
   - IP封禁 → 移除封禁规则
   - 账号锁定 → 解锁账号
   - 进程隔离 → 恢复进程运行
   - 文件隔离 → 还原文件
4. 确认回滚结果

**注意事项：**

- 非所有处置都支持回滚（如文件删除不支持恢复）
- 超过24小时的处置建议评估后再执行回滚
- 回滚操作本身也会记录在审计日志中

## 模块6：威胁狩猎

### 6.1 威胁狩猎概览

威胁狩猎（Threat Hunting）是一种主动的安全分析方法，区别于被动等待告警。SecMind的威胁狩猎模块提供了：

- 假设驱动的狩猎模式
- 跨数据源关联查询
- IOC批量导入和匹配
- 行为分析（UEBA）
- 狩猎结果一键转案件

### 6.2 新建狩猎假设

1. 进入「威胁狩猎」→「新建狩猎」
2. 选择狩猎模板（可选）：
   - 横向移动检测
   - 数据外传检测
   - 异常登录行为
   - 内部信息收集
   - 自定义（自由配置）
3. 配置狩猎条件：
   ```
   狩猎名称：检测异常时间段登录行为
   假设：攻击者可能在非工作时间通过VPN登录内部系统
   
   搜索条件：
   - 事件类型：登录成功
   - 时间范围：22:00 - 06:00
   - 连续天数：>= 3天
   - 源IP不在内部白名单中
   
   数据源范围：VPN日志 / AD登录日志 / 堡垒机日志
   ```
4. 配置狩猎时间范围（最近7天/30天/自定义）
5. 点击「启动狩猎」

### 6.3 IOC导入与匹配

**IOC导入：**

1. 进入「威胁狩猎」→「IOC管理」
2. 支持导入方式：
   - 手动输入单条IOC
   - 批量导入（CSV/JSON/TXT格式）
   - 从威胁情报平台同步
3. 支持IOC类型：
   - IP地址（支持CIDR）
   - 域名
   - URL
   - 文件Hash（MD5/SHA1/SHA256）
   - 邮箱地址
   - 注册表路径
4. 配置IOC标签和置信度

**IOC匹配：**

1. 选择IOC列表，点击「执行匹配」
2. 配置匹配范围（全量数据 / 指定数据源 / 指定时间范围）
3. 系统自动扫描所有数据源中的数据
4. 匹配结果展示：
   ```
   匹配完成：扫描1,234,567条记录
   命中IOC：5条
   
   IOC命中详情：
   ├─ 45.33.32.156 (恶意C2) → 匹配到3条防火墙日志
   │   时间：2026-05-17 14:23:11
   │   源：45.33.32.156:443 → 目标：172.16.0.50:53421
   ├─ evil.com (钓鱼域名) → 匹配到2条DNS日志
   └─ a1b2c3d4... (恶意文件Hash) → 未匹配
   ```
5. 将命中结果一键创建为告警或案件

### 6.4 跨数据源关联

SecMind支持在狩猎过程中跨数据源关联分析：

**跨数据源查询示例：**

```sql-like查询语法
FIND src_ip FROM firewall_log WHERE action=block AND time>2026-05-01
JOIN ad_log WHERE event=login_failed AND src_ip IN result
RETURN src_ip, count(failed_login) AS attempts
WHERE attempts > 100
SORT BY attempts DESC
```

**可视化关联操作：**

1. 在狩猎结果中点击任意IP/域名/资产
2. 选择「关联分析」
3. 系统自动查询所有数据源中与该实体相关的记录
4. 以关联图形式展示实体之间的关系
5. 分析师可在关联图中进一步探索

### 6.5 狩猎结果管理与导出

- 狩猎结果自动保存为「狩猎报告」
- 支持将狩猎发现一键转为告警或案件
- 支持导出狩猎报告（PDF/CSV/JSON）
- 支持狩猎任务定时执行（周期性狩猎）

## 模块7：剧本编排

### 7.1 剧本编辑器概览

SecMind剧本编辑器采用可视化拖拽式操作，支持以下核心概念：

| 概念 | 说明 |
|------|------|
| 触发器 | 剧本的启动条件（告警触发、定时触发、手动触发） |
| 节点 | 编排的基本单元，每个节点执行一个具体操作 |
| 连线 | 定义节点之间的执行顺序和条件 |
| 变量 | 在剧本执行过程中传递的上下文数据 |
| 条件分支 | 根据条件选择不同的执行路径 |
| 并行节点 | 多个节点同时执行 |

### 7.2 创建简单剧本

**场景示例：当检测到暴力破解告警时，自动封禁源IP并通知管理员。**

**操作步骤：**

1. 进入「剧本编排」→「新建剧本」
2. 输入剧本信息：
   - 名称：暴力破解自动封禁
   - 描述：检测到SSH暴力破解告警时，自动封禁源IP并通知安全团队
   - 标签：暴力破解、自动封禁、P2级
3. 配置触发器：
   - 类型：告警触发
   - 匹配条件：告警名称包含"暴力破解" AND 严重级别=高危
   - 数据源：所有
4. 编排节点：

   **节点1：查询告警详情**
   ```
   节点类型：数据查询
   配置：从触发告警中提取源IP、目标IP
   ```

   **节点2：查询威胁情报**
   ```
   节点类型：威胁情报
   配置：查询源IP是否在已知恶意库中
   输出：threat_score
   ```

   **节点3：条件分支**
   ```
   条件：threat_score > 70（高风险）
   ├─ True → 节点4
   └─ False → 节点6
   ```

   **节点4：封禁IP（高风险时执行完整封禁）**
   ```
   节点类型：防火墙处置
   配置：
   - 设备：所有防火墙
   - 动作：添加黑名单
   - 目标：源IP
   - 有效期：48小时
   - 备注：剧本"暴力破解自动封禁"触发
   ```

   **节点5：发送紧急通知**
   ```
   节点类型：通知发送
   配置：
   - 渠道：企业微信 + 邮件
   - 收件人：安全运营组
   - 内容：[紧急] 检测到高风险暴力破解，源IP已自动封禁
   ```

   **节点6：发送普通通知（低风险时仅通知）**
   ```
   节点类型：通知发送
   配置：
   - 渠道：邮件
   - 收件人：值班分析师
   - 内容：检测到低风险暴力破解告警，请人工核查
   ```

5. 点击「验证」检查编排逻辑
6. 验证通过后点击「保存并发布」

### 7.3 剧本测试

1. 在剧本编辑器中点击「测试」
2. 选择测试模式：
   - **模拟测试**：使用模拟数据运行，不触发真实处置
   - **沙盒测试**：在隔离沙盒环境中执行
3. 输入测试数据（模拟告警JSON）
4. 点击「开始测试」
5. 实时查看剧本执行过程：
   ```
   [10:00:01] 触发器匹配成功
   [10:00:02] 节点1：查询告警详情 → 完成 (0.3s)
   [10:00:03] 节点2：查询威胁情报 → 完成 (1.2s) 威胁评分：85
   [10:00:03] 节点3：条件分支 → threat_score>70 为 True
   [10:00:04] 节点4：封禁IP → 暂不执行（测试模式）
   [10:00:04] 节点5：发送通知 → 暂不执行（测试模式）
   [10:00:04] 测试完成，共执行4个节点，耗时1.8秒
   ```
6. 根据测试结果调整剧本配置

### 7.4 剧本发布与监控

**发布流程：**

1. 测试通过后，点击「发布」
2. 选择发布环境：测试环境 → 预发布环境 → 生产环境
3. 填写发布说明
4. 确认发布

**运行监控：**

进入「剧本编排」→「运行记录」，可查看：

| 字段 | 说明 |
|------|------|

### 9.1 告警过滤规则

```
  条件：datasource=DNS AND dst_ip IN 10.0.0.0/8
  操作：隐藏

规则4：关注关键资产告警
  条件：dst_ip IN [核心资产IP列表] AND severity >= 高危
```

## 模块10：实训演练

### 演练1：模拟告警分析完整流程

**场景描述：**

你的仪表盘显示有1条高危告警等待处理。告警详情如下：

```
告警名称：来自外部的RDP暴力破解成功
源IP：203.0.113.45
目标IP：10.10.10.100（资产：DC-01域控服务器）
时间：2026-05-18 02:15:30
严重级别：高危
原始告警：源IP在1小时内尝试RDP登录500次，于02:15:15成功登录
AI状态：已完成分析，置信度88
```

**操作要求：**

1. **接收告警**：
   - 在告警列表中找到该告警
   - 查看告警详情
   - 阅读AI研判结果

2. **分析告警**：
   - 查看AI推理依据是否充分
   - 查看该源IP的历史活动记录
   - 查看目标资产DC-01的敏感度
   - 判断是否为真实威胁

3. **决策操作**：
   - 如确认真实告警，执行确认操作
   - 添加分析备注（说明判断理由）

4. **触发AI调查**：
   - 选择"标准"调查模式
   - 等待调查完成（模拟环境中约为3-5分钟）

5. **复核AI调查结论**：
   - 阅读调查时间线和攻击链路图
   - 确认受影响资产范围
   - 查看证据列表
   - 同意或修正AI的处置建议

6. **创建案件**：
   - 将告警关联到新建案件
   - 设置案件严重级别为"高危"
   - 填写案件描述

7. **执行处置**：
   - 执行IP封禁（防火墙上封禁203.0.113.45）
   - 执行账号审查（检查DC-01上的管理员账号）
   - 记录处置结果

8. **结案**：
   - 确认所有处置措施已执行完成
   - 填写结案报告
   - 结案

**验收标准：**

- 告警状态变更为"已确认"
- AI调查完成并生成报告
- 案件状态为"已结案"
- 处置记录完整可追溯

### 演练2：AI调查与深度分析

**场景描述：**

你收到一条告警：「内部主机向已知恶意域名发起DNS查询」。告警详情：

```
告警名称：DNS查询已知恶意域名
源IP：10.10.20.50（资产：DEV-Web-03）
恶意域名：evil-c2.xyz（关联APT29）
时间：2026-05-18 08:30:00 - 持续进行中
严重级别：紧急
AI状态：已完成，置信度95
```

**操作要求：**

1. **触发深度调查**：
   - 选择"深度"调查模式
   - 启用外部威胁情报
   - 调查范围选择全量关联信息

2. **分析调查结果**：
   - 查看攻击时间线，确认感染时间点
   - 分析C2通信模式和频率
   - 检查是否有数据外传迹象
   - 识别受影响的周边资产

3. **绘制攻击链路**：
   - 在调查报告中查看AI生成的攻击链路图
   - 补充AI可能遗漏的关联信息
   - 更新攻击链路

4. **制定处置方案**：
   - 基于调查结果，制定完整的处置计划
   - 包含短期阻断措施和长期修复建议

5. **撰写调查报告**：
   - 编写完整的分析报告
   - 包含事件摘要、时间线、影响范围、处置建议
   - 将报告添加为案件的证据

**验收标准：**

- AI调查报告完整（含时间线、攻击链路、证据）
- 人工补充不少于2条额外发现
- 处置方案覆盖短期和长期措施
- 分析报告结构完整

### 演练3：威胁狩猎实践

**场景描述：**

安全团队怀疑有APT组织正在针对公司内部网络进行潜伏渗透。你需要主动进行威胁狩猎。

**操作要求：**

1. **建立狩猎假设**：
   - 假设：攻击者可能通过鱼叉邮件投递恶意文档，建立持久化后门
   - 定义搜索条件：检测office进程启动powershell、wmic等异常子进程

2. **配置狩猎条件**：
   ```
   搜索条件：
   - 父进程：WINWORD.EXE / EXCEL.EXE / OUTLOOK.EXE
   - 子进程：powershell.exe / cmd.exe / wmic.exe / mshta.exe
   - 时间范围：最近14天
   - 数据源：EDR日志
   ```

3. **执行狩猎任务**：
   - 启动狩猎
   - 等待狩猎完成

4. **分析狩猎结果**：
   - 审查命中的可疑事件
   - 逐一分析每个事件的上下文
   - 识别真正的威胁

5. **IOC匹配**：
   - 导入提供的IOC列表（恶意域名、IP、Hash）
   - 执行IOC匹配
   - 确认匹配结果

6. **将发现转为案件**：
   - 将确认真实威胁的发现创建为案件
   - 进入标准处置流程

**验收标准：**

- 狩猎任务成功完成
- 从结果中识别出至少1个真实威胁
- IOC匹配至少发现1条命中
- 创建的案件信息完整

### 演练4：剧本编排与自动化响应

**场景描述：**

你需要创建一个自动化响应剧本：当检测到"敏感文件外传"告警时，自动阻断连接、标记涉及账号并通知安全团队。

**操作要求：**

1. **设计剧本流程**：
   ```
   触发器：告警触发（敏感文件外传）
   ├─ 节点1：提取告警中的源IP、目标IP、账号、文件信息
   ├─ 节点2：查询账号权限等级
   ├─ 节点3：条件分支（账号为管理员→高优先级 / 普通用户→普通优先级）
   │   ├─ 高优先级：节点4a 阻断连接 + 节点5a 禁用账号 + 节点6a 紧急通知
   │   └─ 普通优先级：节点4b 阻断连接 + 节点5b 通知管理员
   ```

2. **配置触发器条件**：
   - 告警名称包含"数据外传"或"敏感文件外传"
   - 严重级别 >= 高危

3. **配置各节点参数**：
   - 防火墙阻断节点：配置目标为源IP，有效期4小时
   - 账号操作节点：禁用/标记账号
   - 通知节点：配置通知模板和收件人

4. **测试剧本**：
   - 使用模拟告警数据测试
   - 验证条件分支逻辑正确
   - 确认每个节点的输出符合预期

5. **发布剧本**：
   - 通过测试后发布剧本
   - 启用自动触发

**验收标准：**

- 剧本测试通过，所有节点执行正确
- 条件分支正确区分高优先级和普通优先级
- 通知内容包含完整的告警信息和处置结果
- 剧本发布后处于启用状态
