# FAQ常见问题

## 一、产品与技术类

### 1. SecMind与其他SOC平台有什么区别？

SecMind AI安全运营平台区别于传统SOC平台的五大核心优势：

1. **AI原生架构**：SecMind从底层构建时就以AI为核心，而非在传统SOC上叠加AI功能。内置的大语言模型（LLM）和安全推理引擎能够自主进行告警研判、调查分析和处置建议生成。
2. **自主调查能力**：AI agent可以像人类分析师一样自主展开调查，逐层深挖关联线索，生成完整调查推理报告，大幅缩短MTTR降低MTTR（平均响应时间）。
3. **端到端自动化**：从告警接入、AI分析、案件生成到自动化处置，形成完整的闭环流程，无需多平台切换。
4. **多租户与RBAC**：原生支持多租户架构和细粒度RBAC权限控制，适用于MSSP（托管安全服务提供商）和企业多子公司场景。
5. **低代码剧本编排**：通过可视化拖拽方式编排安全响应剧本，降低运维门槛，无需编写复杂脚本。

### 2. AI准确率有多高？会误判吗？

SecMind AI的告警研判准确率在实测环境中达到**95%以上**（依据数百万条真实告警数据的测试结果）。但需要说明以下几点：

- **准确率指标**：我们采用精确率（Precision）、召回率（Recall）和F1分数三个维度综合评估，其中真实告警（True Positive）识别准确率持续保持在93%-97%之间。
- **误判说明**：任何AI系统都无法做到100%准确。SecMind提供了**置信度评分**机制，每条AI研判结果都会附带0-100的置信度分数，供分析师参考。
- **人工复核机制**：对于置信度低于阈值（默认80分）的告警，系统会自动标记为"待人工复核"，不会直接处置。
- **持续优化**：系统支持人工反馈闭环——分析师对AI结果的确认或驳回操作会反馈给模型，持续提升推理精度。

### 18. 告警去重和降噪是怎么实现的？

平台采用多层告警降噪机制：

- **第一层（接入层）**：基于规则的告警过滤，支持按源IP、告警类型、严重级别等条件过滤无效告警。
- **第二层（归并层）**：基于相似度算法的告警归并，将相同源/目标的同类告警合并为告警事件束。
- **第三层（AI层）**：AI引擎对归并后的告警事件进行语义理解和上下文分析，判断是否为真实威胁。
- **第四层（关联层）**：通过攻击链路分析，将属于同一攻击链的告警自动关联，避免重复告警。

通过这四层机制，平均告警压缩比可达**50:1至200:1**，大幅减少分析师需要处理的告警量。

## 三、安全与合规类

### 1. 客户数据如何保护？

SecMind采取多层次数据保护措施：

- **传输加密**：所有网络通信强制使用TLS 1.2/1.3加密
- **存储加密**：数据落盘采用AES-256加密，密钥由HSM（硬件安全模块）或KMS（密钥管理服务）管理
- **访问控制**：基于RBAC的细粒度权限控制，最小权限原则
- **数据隔离**：多租户环境下，每个租户的数据逻辑隔离，通过租户ID路由确保数据不串越
- **审计追踪**：所有数据访问操作记录完整审计日志
- **数据脱敏**：支持对敏感字段（如IP地址、用户名、证书信息）进行动态脱敏展示
- **防泄露**：内置DLP策略，防止未经授权的数据导出

### 2. 通过哪些安全认证？

SecMind已完成以下安全认证和合规评估：

- **等保三级**：通过国家网络安全等级保护三级测评
- **ISO 27001**：信息安全管理体系认证
- **ISO 27701**：隐私信息管理体系认证
- **SOC 2 Type II**（进行中）：服务组织控制报告
- **《个人信息保护法》（PIPL）合规**：已完成合规评估

### 3. AI模型的训练数据来源于哪里？

SecMind AI模型的训练数据来源于：

- **公开安全数据集**：CICIDS、UNSW-NB15、NSL-KDD等学术界公开数据集
- **开源威胁情报**：MITRE ATT&CK、CVE、NVD等公开安全知识库
- **脱敏的聚合数据**：经客户授权并完全去标识化后的告警数据（客户可自主选择是否参与）
- **合成数据**：基于真实攻击模式生成的模拟攻击数据
- **安全研究团队**：团队手动标注的高质量安全分析样本

**重要说明**：任何情况下，客户的原始告警数据不会直接用于模型训练。私有化部署场景中，所有数据留存于客户本地。

### 4. 数据存储在哪里？

- **私有化部署**：数据存储于客户指定的数据中心或私有云环境，平台不向外部传输任何告警数据（除可选的威胁情报查询外）。

### 6. 是否支持数据加密密钥由客户管理？

支持。私有化部署客户可选择以下密钥管理模式：

- **平台管理密钥**：由平台内置KMS管理密钥，操作简便
- **客户管理密钥（BYOK）**：客户使用自己的HSM或KMS（如阿里云KMS、AWS KMS、自建HashiCorp Vault）管理加密密钥，平台通过调用加密API进行加解密操作。即使平台被攻破，也无法解密客户数据也无法被解密。
- **客户提供密钥（HYOK）**：客户完全掌控加密密钥，所有加解密操作在客户环境中完成。

### 7. 发生安全事件如何响应？

当发现平台自身的安全事件时，我们执行以下流程：

1. **发现与报告**：通过平台自监控系统或安全团队发现、外部报告
2. **评估与分级**：30分钟内完成事件定级（P0-P4）
3. **响应处置**：P0事件（严重）15分钟内启动应急响应，P1事件（高危）30分钟内启动
4. **客户通知**：P0/P1事件在1小时内通知受影响客户
5. **事后复盘**：72小时内输出事件报告，包括根因、影响范围和改进措施

- **渗透测试**：每季度由第三方安全团队进行渗透测试
- **漏洞响应**：关键漏洞24小时内修复，高危漏洞72小时内修复
- **CVE跟踪**：持续跟踪与平台相关组件CVE公告，及时进行安全更新
