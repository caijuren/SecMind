## 一、协议概述

### 1.1 协议目的

本服务等级协议（Service Level Agreement，以下简称"SLA"）旨在明确SecMind AI安全运营平台（以下简称"SecMind"或"平台"）向客户承诺的服务质量指标、可用性保障、技术支持标准以及服务不可用时的补偿方案。

### 1.2 适用版本

本SLA适用于SecMind平台以下版本：

| 版本名称 | 适用对象 | 典型场景 |
|---------|---------|---------|
| 免费版（Free） | 个人安全研究者、小型团队试用 | 功能评估、PoC验证 |
| 专业版（Professional） | 中型企业SOC团队 | 日常安全运营使用 |
| 企业版（Enterprise） | 大型企业、金融机构、政务单位 | 生产环境核心安全运营 |

### 1.3 定义

- **服务可用性**：客户能够正常访问和使用SecMind平台核心功能的时间比例。
- **月度**：自然月，从每月第1天00:00:00至最后一天23:59:59（北京时间，UTC+8）。
- **计划维护**：SecMind提前通知客户的计划内系统维护时间窗口。
- **服务不可用**：客户无法正常使用平台核心功能，且非客户侧原因或除外情形所致。

| 等级 | 标签 | 定义 | 典型示例 |

- 确认后30分钟内创建事件工单并通知客户

- 故障修复后24小时内提交故障报告

**P3故障（中）：**
- 确认后创建事件工单
- 每个工作日同步一次处理进展
- 修复后在工单中更新处理结论

**P4故障（低）：**
- 确认后创建工单或记录
- 在修复或答复后关闭时更新处理结论

## 八、月度可用性计算公式

### 8.1 可用性计算公式

```
月度服务可用性 = (总分钟数 - 不可用分钟数) / 总分钟数 × 100%
```

其中：
- **总分钟数** = 当月自然天数 × 24 × 60
- **不可用分钟数** = 当月内所有服务不可用持续时间的总和（以分钟为单位）
- 服务不可用起始时间从SecMind确认故障或客户报告且SecMind确认故障的时间开始计算
- 服务不可用结束时间为故障恢复、服务恢复正常的时间

### 8.2 计算示例

假设某自然月有30天，总分钟数为 30 × 24 × 60 = 43,200分钟。

当月发生一次P1故障持续90分钟，一次P2故障持续120分钟：
```
不可用分钟数 = 90 + 120 = 210分钟
月度服务可用性 = (43,200 - 210) / 43,200 × 100% = 99.514%
```

对于企业版客户（承诺99.99%），当月可用性未达标。
对于专业版客户（承诺99.9%），99.514% > 99.9%，达标。

### 8.3 可用性监控和报告

- SecMind使用内部和第三方监控工具持续测量服务可用性
- 每月结束后5个工作日内，SecMind向专业版和企业版客户提供上月的月度可用性报告
- 客户可随时可在平台"系统监控"页面查看当前和历史服务状态

## 九、服务不可用的除外情形

以下情形不计入服务不可用时间，SecMind不对此类情形下的服务中断承担SLA责任：

### 9.1 计划维护

- SecMind提前通知的计划内系统升级、维护和变更
- 免费版：无提前通知要求
- 专业版：提前48小时通知，每月总维护时间不超过4小时
- 企业版：提前7天通知，每月总维护时间不超过2小时
- 紧急安全补丁维护不受上述时间限制，但SecMind会尽最大努力缩小维护窗口并及时通知客户

### 9.2 不可抗力

因以下不可抗力因素导致的服务中断：

- 自然灾害：地震、洪水、台风、火灾等
- 社会事件：战争、恐怖袭击、暴乱、罢工等
- 公共卫生事件：重大疫情、隔离等
- 政府行为：法律、法规、政策变更，政府管制等
- 电力供应中断（非SecMind原因）
- 网络主干中断（非SecMind原因）

### 9.3 第三方服务故障

- 云服务商基础设施故障（如阿里云ECS、RDS、OSS等服务大规模故障）
- AI模型API服务商服务中断（如OpenAI、DeepSeek、阿里云等API服务不可用）
- 客户依赖的第三方集成服务故障（如客户自建SIEM、EDR等系统的接口异常）
- CDN服务故障导致的静态资源加载异常

### 9.4 客户侧原因

- 客户未按使用规范操作导致的故障
- 客户网络环境问题导致无法访问平台
- 客户账户欠费、到期或违规使用被限制
- 客户未及时更新客户端或集成配置
- 客户自行修改系统变更导致的兼容性问题
- 客户超过使用配额限制导致的限流

### 9.5 其他除外情形

- 实验性功能和Beta版本功能（明确标注为Beta的功能不纳入SLA）
- 非生产环境（如开发环境、测试环境、Demo环境）
- 因安全攻击导致的必要防护措施（如DDoS清洗、WAF拦截等），前提是SecMind采取了合理的安全防护措施
- 因法律法规或政府要求导致的必要限制

## 十、服务补偿方案

### 10.1 补偿条件

当月度服务可用性低于对应版本的承诺标准时，客户有权申请服务补偿。

| 版本 | 承诺可用性 | 补偿触发条件 | 补偿形式 |
|------|-----------|-------------|---------|
| 专业版 | 99.9% | 月度可用性 < 99.9% | 服务积分或延长服务期 |
| 企业版 | 99.99% | 月度可用性 < 99.99% | 服务积分或延长服务期或退款 |

免费版不提供SLA服务补偿。

### 10.2 补偿标准

| 月度可用性 | 专业版补偿 | 企业版补偿 |
|-----------|-----------|-----------|
| ≥ 99.9% | 无补偿 | - |
| ≥ 99.5% 且 < 99.9% | - | 当月月度服务费的10% |
| ≥ 99.0% 且 < 99.5% | - | 当月月度服务费的10% |
| ≥ 99.99% | - | 无补偿 |
| ≥ 99.9% 且 < 99.99% | - | 当月月度服务费的10% |
| ≥ 99.5% 且 < 99.9% | 当月月度服务费的10% | 当月月度服务费的20% |
| ≥ 99.0% 且 < 99.5% | 当月月度服务费的15% | 当月月度服务费的30% |
| < 99.0% | 当月月度服务费的25% | 当月月度服务费的50% |

### 10.3 补偿上限

- 任何单个月度的补偿金额不超过该月度服务费的50%
- 任何12个月周期内的累计补偿金额不超过该周期内服务费总额的100%
- 补偿仅针对受直接影响的服务费，不包含附加服务或一次性费用

### 10.4 补偿申请流程

1. **申请时间**：客户需在故障发生的当月起90天内提交补偿申请
2. **申请渠道**：通过在线工单系统提交，或联系客户经理
3. **审核时间**：SecMind在收到申请后10个工作日内完成审核
4. **补偿发放**：审核通过后，补偿在30天内以服务积分或延长服务期形式发放

### 10.5 补偿形式说明

- **服务积分**：可用于抵扣后续服务费用，有效期12个月
- **延长服务期**：按补偿金额等值延长服务期限
- **退款**：仅适用于企业版客户，且仅限于年度预付场景

## 十一、客户责任义务

### 11.1 合理使用

### 11.2 及时报告

- **故障报告**：客户在发现服务异常时应及时通过指定的支持渠道向SecMind报告
- **安全事件**：客户发现涉及平台安全的可疑行为或安全事件时应立即通知SecMind
- **反馈机制**：客户应积极反馈使用中遇到的问题，帮助SecMind持续改进服务质量

### 11.3 配合排查

- **信息提供**：SecMind在排查故障时，客户应积极配合提供必要的日志、截图、时间点等信息
- **环境配合**：客户应在合理范围内配合SecMind进行故障复现和诊断测试
- **变更通知**：客户进行可能影响服务使用的重大变更（如网络配置、集成变更等）时，应提前通知SecMind

### 11.4 安全责任

- **账户安全**：客户应妥善保管账户凭证，对使用其账户进行的所有活动负责
- **密码管理**：客户应设置强密码并定期更换，启用MFA多因素认证
- **权限管理**：客户应定期审查和清理用户权限，确保最小权限原则
- **数据备份**：客户应根据自身需要对其重要数据进行备份（SecMind提供数据保护，但客户应制定完整的业务连续性计划）

### 11.5 第三方集成

- **接口可用性**：客户自行集成的第三方系统接口的可用性由客户负责
- **数据安全**：客户通过集成接口传输的数据安全由客户和对应的第三方服务商负责
- **兼容性**：客户应关注集成接口的版本变更和兼容性通知
