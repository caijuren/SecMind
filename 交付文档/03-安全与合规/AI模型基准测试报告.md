## 一、测试目的与测试目的与范围

### 1.1 测试目的

本次基准测试旨在系统性地评估SecMind AI安全运营平台当前支持的多款大语言模型在核心安全分析场景下的表现，为模型选型、路由策略优化和成本控制提供数据支撑。具体目标包括：

1. 量化评估各模型在告警分析、攻击链还原、处置建议生成等安全场景下的准确性和可靠性
2. 对比各模型在中文安全场景下的理解能力和推理质量
3. 评估各模型的响应速度，为实时性要求不同的场景提供选型依据
4. 构建成本效益评估框架，支持在保障质量的前提下优化运营成本
5. 为SecMind模型路由引擎（ModelRouter）的智能调度策略提供数据基础

### 1.2 测试范围

| 维度 | 内容 |
|------|------|
| 测试模型 | GPT-4o、DeepSeek V3、通义千问 Qwen-Max、GLM-4 |
| 测试场景 | 告警误报判断、攻击链还原、处置建议生成、IOC提取、多语言告警分析 |

## 二与测试场景设计

### 2.1 场景一：告警误报判断（Alert Triage）

**任务描述**：给定一条告警的完整信息（告警类型、源IP、目的IP、告警描述、触发规则等），模型需判断该告警是否为真实威胁（True Positive）或误报（False Positive）。

**评估方式**：与安全专家标注结果对比，计算二元分类指标。

**示例告警（脱敏）：**
```
告警类型：C2通信检测
告警描述：内网主机 10.88.31.42 向外部IP 198.51.100.23（位于俄罗斯）发起
了频率为每5分钟一次的HTTPS连接，连接时间为凌晨02:00-04:00，
符合已知C2心跳行为模式（MITRE ATT&CK T1043）。
触发规则：C2_Beaconing_Detection_v2
```

**期望输出：** 是否为真实威胁（是/否）+ 理由说明 + 置信度评分

### 2.2 场景二：攻击链还原（Attack Chain Reconstruction）

**任务描述**：给定一组时间关联的多条告警，模型需还原攻击者的完整攻击链（Kill Chain），识别每个阶段对应的告警，并对攻击意图进行总结分析。

**评估方式**：人工评分（1-5分），考察攻击链还原的完整性还原度、阶段划分准确性、证据关联合理性。

**告警序列示例（脱敏）：**
```
1. [09:15] 告警：外部扫描探测 - IP 203.0.113.5 扫描端口 22,443,8080,3306
2. [09:23] 告警：SSH暴力破解 - 203.0.113.5 尝试登录 10.88.31.50 (失败12次)
3. [09:28] 告警：SSH登录成功 - 203.0.113.5 登录 10.88.31.50 (非工作时间)
4. [09:35] 告警：可疑进程启动 - 10.88.31.50 启动进程 /tmp/update (未知MD5)
5. [09:42] 告警：C2外联 - 10.88.31.50 连接 45.33.32.156:443
```

### 2.3 场景三：处置建议生成（Response Recommendation）

**任务描述**：根据告警分析结果和攻击链还原结论，模型需生成具体的处置建议，包括立即措施、后续跟进和长期防护建议。

**评估方式**：人工评分（1-5分），考察建议的可操作性、完整性、安全最佳实践符合度。

### 2.4 场景四：IOC提取（Indicator of Compromise Extraction）

**任务描述**：从非结构化安全报告或威胁情报文本中，自动提取IOC指标（IP地址、域名、URL、文件哈希、邮箱地址等），并按类型分类结构化输出。

**评估方式**：精确率、召回率、F1 Score，与人工标注的IOC列表对比。

### 2.5 ：多语言告警分析（Multilingual Alert Analysis）

**任务描述**：分析中英文混合的告警内容（常见于国内企业环境，安全设备告警用英文、描述含中文注释），判断威胁严重程度和影响范围。

**评估方式**：人工评分（1-5分），考察语言理解准确性、关键信息提取完整度。

## 四与评估指标体系

### 4.1 核心评估指标

#### 4.1.1 精确率（Precision）

```
Precision = TP / (TP + FP)
```

衡量模型识别结果的准确性，即模型判定为真的样本中，确为真的比例。对于告警误报判断场景，精确率反映"模型判定为真实攻击的结果中，实际为真实攻击的比例"。

#### 4.1.2 召回率（Recall）

```
Recall = TP / (TP + FN)
```

衡量模型对真实威胁的覆盖能力，即所有真实为正的样本中，模型正确识别出的比例。

#### 4.1.3 F1 Score

```
F1 Score = 2 × (Precision × Recall) / (Precision + Recall)
```

精确率和召回率的调和平均值，综合衡量模型的分类性能。

#### 4.1.4 人工评分（Human Rating）

对于攻击链还原、处置建议生成等需要主观评定的场景，由3名资深安全分析师独立对模型输出进行1-5分评分，取平均值作为最终评分标准。

| 评分 | 含义 |
|------|------|
| 5分 | 完全正确，逻辑严密，可直接使用 |
| 4分 | 基本正确，小幅偏差可接受 |
| 3分 | 部分正确，存在明显问题需人工修正 |
| 2分 | 大部分错误，仅少量有用信息 |
| 1分 | 完全不相关或严重误导 |

#### 4.1.5 响应时间（Latency）

测量从API请求发出到收到完整响应的时间，分别统计P50（中位数）、P95（95分位）和P99（99分位）延迟。

#### 4.1.6 成本（Cost）

统计每次API调用的Token消耗和费用，按模型提供方的标准单价计算。计费模型如下：

| 模型 | 输入价格（每1K tokens） | 输出价格（每1K tokens） |

## 六与中文场景下国产模型表现分析

### 6.1 国产模型表现亮点

本次测试中，国产模型（DeepSeek V3, 通义千问 Qwen-Max, GLM-4）在以下方面表现突出：

**6.1.1 DeepSeek V3 - 性价比之王**

DeepSeek V3在本次测试中展现出令人瞩目的综合实力：

| 维度 | 表现 | 与GPT-4o差距 | 优势 |
|------|-------------|------|
| 综合F1 Score | -0.027（2.9%差距） | 仅为GPT-4o成本的2.6% |
| 综合人工评分 | -0.14（3.1（3.1%差距） | 响应速度是GPT-4o的2.4倍 |
| 中文场景评分 | -0.05（1.1%差距） | 中文安全场景理解接近GPT-4o顶级水平 |

DeepSeek V3在以下场景表现特别抢眼：
- **告警误报判断**：F1 Score 0.913，接近GPT-4o的0.940
- **IOC提取**：F1 Score 0.943，几乎与GPT-4o持平
- **多语言告警分析**：人工评分4.45，仅低于GPT-4o 0.05分

**6.1.2 通义千问 Qwen-Max - 中文场景优势**

通义千问 Qwen-Max在纯中文告警分析场景下表现优异：

- 中文安全术语理解准确度高，对国内安全设备厂商（如奇安信、深信服、绿盟等）的告警格式识别良好
- 对国内合规要求理解准确（等保2.0相关术语识别）
- 在处理国内特有的安全威胁（如钓鱼攻击中涉及的国内社交工程话术）时表现自然

**6.1.3 GLM-4 - 稳定可靠**

GLM-4在各项测试中表现稳定，虽然在顶尖指标上略有不及其他模型，但胜在：
- 输出一致性好，在重复测试中结果波动最小
- 对指令遵循度高，格式化输出表现最好
- 支持128K超长上下文，适合长文本安全报告分析

### 6.2 中文安全领域特定术语理解测试

测试了各模型对安全领域专业术语的理解准确性：

| 安全术语 | GPT-4o | DeepSeek V3 | 通义千问 Qwen-Max | GLM-4 |
|---------|--------|-------------|-------------------|-------|
| MITRE ATT&CK框架映射 | 5.0/5.0 | 4.8/5.0 | 4.5/5.0 | 4.3/5.0 |
| 国内安全产品告警格式 | 4.0/5.0 | 4.7/5.0 | 4.8/5.0 | 4.5/5.0 |
| 等保2.0合规要求 | 3.5/5.0 | 4.5/5.0 | 4.7/5.0 | 4.3/5.0 |
| 中文威胁情报解析 | 4.2/5.0 | 4.8/5.0 | 4.6/5.0 | 4.4/5.0 |

**结论：** 国产模型在国内安全生态的特定场景下具有显著理解优势，特别是对国内安全工具告警格式、等保合规要求等方面的理解和处理能力均超越GPT-4o。

## 九与结论与选型建议

### 9.1 核心发现

1. **GPT-4o**在各项安全分析任务中均保持最高质量，特别是在复杂推理场景（攻击链还原、处置建议生成）中优势明显，但成本最高、延迟最长。
2. **DeepSeek V3**在本次测试中展现出极高的性价比，质量接近GPT-4o（综合差距不到3%），但成本仅为GPT-4o的2.2%，延迟仅为GPT-4o的41.7%。在中文安全场景下，DeepSeek V3的表现接近GPT-4o水平，是国产模型中最优选择。
3. **通义千问 Qwen-Max**在纯中文和国内安全生态场景中表现优异，对国内安全厂商设备告警格式的理解和识别能力突出。
4. **GLM-4**表现稳定，长上下文支持和输出一致性是其亮点，适合对格式规范性要求高的场景。

### 9.2 推荐选型矩阵

| 攻击链还原/深度调查 | GPT-4o | DeepSeek V3 | GPT-4o |
| 处置建议生成 | GPT-4o | DeepSeek V3 | DeepSeek V3 |
| IOC提取 | GPT-4o | DeepSeek V3 | DeepSeek V3 |

| 长文本报告分析 | GPT-4o | GLM-4 | GPT-4o |
| 实时告警流处理 | DeepSeek V3 | DeepSeek V3 | DeepSeek V3 |
| 合规评估与映射 | 通义千问 | DeepSeek V3 | 通义千问 |

### 10.4 测试方法

1. **批量执行**：所有测试样本通过测试框架批量提交至各模型API，记录完整请求响应链路数据
2. **结果收集**：自动收集每次调用的输出内容、Token用量、延迟时间、错误信息
3. **自动评估**：结构化任务（告警误报判断、IOC提取）通过脚本自动对比标注结果计算指标
4. **人工评估**：非结构化任务（攻击链还原、处置建议生成、多语言告警分析）由标注团队进行人工评分
5. **交叉验证**：10%的样本由两名分析师独立重复评分，确保评分一致性

### 10.5 局限性说明

- 本测试结果基于特定时间点（2026年4月）的各模型API版本，不排除后续版本更新带来性能变化
- 测试网络环境为阿里云华东2区，对不同地理位置的客户实际体验可能存在网络延迟差异
- 测试数据集以中英文安全告警为主，不覆盖所有可能的安全分析场景
- 模型的经济性分析基于公开定价，实际用量可能因Token定价调整而变化
