# 态势大屏

态势大屏（Situation Room）是 SecMind v3.0 的核心可视化模块，提供安全运营态势的全局可视化指挥中心。

## 功能概述

态势大屏定位为安全运营态势的全局可视化指挥中心，适用于以下场景：

- **安全运营中心大屏展示**：实时呈现整体安全态势，帮助安全团队快速掌握全局
- **管理层汇报**：以直观的数据可视化方式向管理层展示安全运营成效
- **应急指挥**：在安全事件应急响应过程中提供实时态势感知和决策支持

## 核心组件

### 1. 攻击地图 (AttackMap)

基于 Canvas 的世界地图渲染组件，实时展示全球攻击态势。

- **地图渲染**：使用 Canvas API 绘制高分辨率世界地图，支持缩放和平移
- **攻击弧线**：攻击源到目标的贝塞尔曲线弧线，动态展示攻击流向
- **脉冲光晕**：攻击弧线末端带有脉冲光晕效果，颜色按严重程度编码：
  - `critical`：红色 `#FF4444`
  - `high`：橙色 `#FF8800`
  - `medium`：黄色 `#FFCC00`
  - `low`：青色 `#00CCFF`
- **脉冲动画**：新攻击出现时触发放大脉冲动画，吸引注意力
- **响应式设计**：自适应容器大小，支持窗口尺寸变化

### 2. 安全评分仪表盘 (SecurityScoreGauge)

大型半圆仪表盘组件，直观展示当前安全评分。

- **仪表盘结构**：180 度半圆弧形，刻度范围为 0-100
- **渐变色弧**：四段渐变色弧：
  - 0-25：红色（危险）
  - 25-50：橙色（风险）
  - 50-75：黄色（一般）
  - 75-100：青色（安全）
- **指针动画**：白色指针带平滑过渡动画，数值变化时流畅摆动
- **评级标签**：根据当前评分显示对应评级：
  - 75-100：安全
  - 50-75：一般
  - 25-50：风险
  - 0-25：危险
- **趋势箭头**：显示评分变化趋势，上升（↑）、下降（↓）或持平（→）

### 3. 威胁统计面板 (ThreatStatsPanel)

2x2 网格布局的统计卡片组，汇总关键安全指标。

- **活跃威胁数**：当前正在活跃的威胁数量，红色高亮
- **已阻断攻击数**：累计已成功阻断的攻击数量，绿色显示
- **MTTR 平均响应时间**：平均威胁检测和响应时间（单位：分钟），黄色显示
- **告警降噪率**：AI 降噪后的有效告警占比（百分比），青色显示
- **数字动画计数器**：数值变化时触发数字滚动动画
- **趋势指示器**：每张卡片右下角显示短期趋势方向

### 4. 威胁播报 (ThreatTicker)

底部滚动威胁播报条，实时滚动展示最新威胁信息。

- **滚动方式**：从右至左无缝滚动
- **信息格式**：每条播报包含以下字段：
  - 时间戳（HH:mm:ss）
  - 严重级别图标（红/橙/黄/青）
  - 威胁类型名称
  - 简要描述
- **渐变遮罩**：左右两侧设置渐变遮罩，实现淡入淡出效果
- **交互暂停**：鼠标悬停时暂停滚动，便于查看详情

## 布局说明

态势大屏采用 12 列 5 行网格布局：

```
┌─────────────────────────────────────────────────────────────┐
│  攻击地图 (AttackMap)    │ 安全评分 (ScoreGauge) │ 威胁统计  │
│        8列               │       2列            │   2列     │
├─────────────────────────────────────────────────────────────┤
│  告警趋势 (AlertTrend)   │ MITRE 矩阵           │ 攻击类型   │
│        6列               │       3列            │   3列     │
├─────────────────────────────────────────────────────────────┤
│              威胁播报 (ThreatTicker) 12列                   │
└─────────────────────────────────────────────────────────────┘
```

- **顶行**：攻击地图（8列）| 安全评分（2列）| 威胁统计（2列）
- **中行**：告警趋势（6列）| MITRE 矩阵（3列）| 攻击类型分布（3列）
- **底行**：威胁播报（全宽 12列）

## 使用方式

### 自动刷新

态势大屏默认每 5 秒自动刷新全部数据，保持信息实时性。数据通过 WebSocket 推送更新，无需手动操作。

### 演示模式

演示模式隐藏浏览器 UI 框架（地址栏、标签栏等），仅显示数据面板内容，适合墙面大屏展示。

```javascript
// 通过 URL 参数启用演示模式
// 在浏览器地址栏访问：
https://secmind.company.com/situation-room?mode=demo
```

### 手动刷新

点击右上角刷新按钮可立即触发全量数据刷新，无需等待自动刷新周期。

## 最佳实践

1. **大屏展示建议使用 4K 分辨率**：4K（3840×2160）分辨率可完整呈现所有组件，避免信息拥挤
2. **演示模式适合墙面大屏**：演示模式去除浏览器装饰元素，适合安全运营中心墙面大屏长期展示
3. **数据源需配置 WebSocket 实时推送**：为确保态势大屏的实时性，后端数据源应配置 WebSocket 推送机制，避免使用轮询方式
4. **合理配置告警级别过滤**：根据实际运营需求配置告警级别展示策略，避免低级别告警淹没关键信息
5. **定期校准评分模型**：安全评分模型的权重和阈值应定期根据实际运营效果进行校准
