# AI 智能分析

SecMind 的 AI 智能分析功能基于大语言模型，为安全分析师提供告警研判、威胁分析和智能建议，大幅提升安全运营效率。

## 功能概述

AI 智能分析模块提供以下核心能力：

- **告警自动研判**：对入站告警进行自动分析，评估威胁等级和影响范围
- **攻击链还原**：基于多源数据自动还原攻击链路
- **智能建议**：根据分析结果生成结构化的响应处置建议
- **自然语言交互**：通过 AI 对话界面进行交互式安全分析
- **上下文关联**：自动关联情报、历史告警和资产信息

## 使用方式

### 告警自动研判

当告警进入系统后，AI 分析引擎会自动触发研判流程：

1. 告警数据标准化和富化
2. 关联情报源查询（VirusTotal、OTX 等）
3. 历史相似告警匹配
4. LLM 推理分析
5. 生成研判结论和建议

### AI 对话分析

通过 AI 对话界面，分析师可以用自然语言进行深度分析：

```text
用户：分析最近 24 小时内来自 192.168.1.100 的所有告警，判断是否存在横向移动行为

AI：基于最近 24 小时的告警数据分析，我发现以下可疑活动模式：

1. 时间线分析：
   - 08:23 - 端口扫描告警（目标：192.168.1.0/24）
   - 09:15 - SSH 暴力破解告警（目标：192.168.1.105）
   - 10:42 - 异常登录告警（源：192.168.1.105）
   - 11:30 - 数据外泄告警（目标 IP：45.33.xx.xx）

2. 横向移动判断：高度可疑
   - 攻击者首先进行网络侦察
   - 随后通过 SSH 暴力破解获取内网主机权限
   - 利用被控主机发起进一步攻击

3. 建议处置：
   - 立即隔离 192.168.1.105
   - 重置相关账户凭证
   - 排查 192.168.1.100 是否已被入侵
```

### 批量分析

支持对多个告警进行批量 AI 分析：

1. 在告警列表中选择目标告警
2. 点击「AI 批量分析」按钮
3. 系统自动进行关联分析和聚类
4. 生成分析摘要和处置建议

## 配置选项

### 分析策略配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `auto_analysis` | 是否自动分析新告警 | `true` |
| `analysis_threshold` | 触发自动分析的告警级别 | `medium` |
| `max_concurrent` | 最大并发分析数 | `5` |
| `context_window` | 上下文时间窗口（小时） | `24` |
| `include_threat_intel` | 是否关联威胁情报 | `true` |
| `include_asset_info` | 是否关联资产信息 | `true` |

### 模型参数配置

```json
{
  "task_type": "alert_triage",
  "model": "gpt-4",
  "temperature": 0.1,
  "max_tokens": 2048,
  "system_prompt": "你是一个专业的安全分析师...",
  "response_format": "json"
}
```

### 提示词模板

SecMind 支持自定义 AI 分析的提示词模板：

```python
# 通过 API 更新提示词模板
PUT /api/ai/prompts/alert_triage
{
  "name": "alert_triage",
  "template": """
  请分析以下安全告警，给出你的研判结论：

  告警信息：{{alert_data}}
  关联情报：{{threat_intel}}
  历史告警：{{related_alerts}}

  请按以下格式输出：
  1. 威胁等级评估
  2. 攻击手法分析
  3. 影响范围评估
  4. 处置建议
  """
}
```

## 最佳实践

1. **合理设置自动分析阈值**：建议从 `medium` 级别开始，避免低级别告警消耗过多 LLM 资源
2. **定期审查 AI 分析结果**：AI 分析结果应作为辅助参考，关键决策仍需人工确认
3. **优化提示词模板**：根据实际场景调整提示词，可显著提升分析质量
4. **监控 LLM 调用成本**：通过模型路由策略，为不同任务选择合适的模型
