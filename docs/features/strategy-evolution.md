# 策略自演化

SecMind 的策略自演化功能通过 AI 驱动的强化学习机制，实现处置策略的持续优化。系统越用越聪明，处置准确率随运营数据积累持续提升。

## 功能概述

策略自演化定位为 AI 驱动的处置策略持续优化系统，其核心价值在于：

- **自适应优化**：基于历史处置反馈自动调整策略参数，减少人工调优成本
- **持续进化**：随着运营数据积累，策略有效性不断提升
- **闭环反馈**：从处置结果中学习，形成"分析 → 处置 → 反馈 → 优化"的完整闭环
- **精准匹配**：根据攻击阶段、资产关键性等上下文动态选择最优处置动作

## 核心机制

### 1. Q-Learning 强化学习

策略自演化基于 Q-Learning 强化学习算法，在持续的交互中学习最优处置策略。

**状态空间**：由四个维度组合定义系统状态

| 维度 | 取值范围 | 说明 |
|------|----------|------|
| 攻击阶段 | `reconnaissance` / `weaponization` / `delivery` / `exploitation` / `installation` / `c2` / `actions` | 当前攻击所处的阶段 |
| 置信度等级 | `high` / `medium` / `low` | AI 分析对告警判定的置信程度 |
| 资产关键性 | `critical` / `high` / `medium` / `low` | 受影响资产的重要级别 |
| 历史成功率 | `>80%` / `60-80%` / `40-60%` / `<40%` | 同类策略在过去 7 天的平均成功率 |

**动作空间**：系统支持的处置动作集合

```
8 种标准处置动作：
  - isolate          : 隔离资产
  - block_ip         : 封禁 IP
  - kill_process     : 终止进程
  - revoke_session   : 撤销会话
  - reset_credential : 重置凭证
  - quarantine_file  : 隔离文件
  - notify_admin     : 通知管理员
  - create_ticket    : 创建工单

特殊动作：
  - no_action        : 不执行处置（适用于误判场景）
```

**奖励函数**：驱动策略优化的核心反馈信号

| 场景 | 奖励值 | 说明 |
|------|--------|------|
| 正确处置 | `+1` | 处置动作准确匹配告警类型，成功阻断威胁 |
| 误判 | `-5` | 处置动作错误（如误隔离正常资产），给予高额惩罚 |
| 响应延迟 | `-0.1/分钟` | 从告警确认到执行处置的延迟时间，每分钟扣减 |

**学习参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 学习率 α | `0.1` | 控制新经验覆盖旧经验的速度 |
| 折扣因子 γ | `0.9` | 平衡即时奖励与未来奖励的权重 |
| 探索率 ε | `0.1`（初始） | 初始探索概率，随迭代逐步衰减至 `0.01` |

### 2. 策略评估

系统定期对每条策略进行多维度的有效性评估，生成适应度评分和等级。

**评分等级**：

| 等级 | 评分范围 | 含义 |
|------|----------|------|
| `excellent` | ≥ 90 | 策略表现优异，处置准确率高 |
| `good` | 75 - 89 | 策略表现良好，可继续使用 |
| `moderate` | 60 - 74 | 策略表现一般，建议关注 |
| `poor` | < 60 | 策略表现较差，需要重新演化 |

**趋势分析**：基于历史评分数据判断策略发展趋势

| 趋势类型 | 判定标准 | 建议措施 |
|----------|----------|----------|
| 持续上升 | 连续 3 次评估评分递增 | 继续观察，暂不干预 |
| 稳定 | 评分波动幅度 < 5% | 维持现有策略 |
| 持续下降 | 连续 3 次评估评分递减 | 触发策略衰退检测 |

### 3. 置信度优化

置信度阈值决定处置策略的自动化执行程度，系统根据运营数据动态调整。

**动态调整机制**：

- **误判率升高**：自动提高置信度阈值，将更多处置转为人工审批
- **漏报率升高**：自动降低置信度阈值，扩大自动处置覆盖范围
- **成功率升高**：适度降低阈值，提升自动化效率

**执行策略**：

```
高置信度（≥ 阈值）→ 自动执行处置
低置信度（< 阈值）→ 生成处置建议，等待人工审批
```

**自适应算法**：

```
new_threshold = base_threshold
  + α_false_positive * false_positive_rate * 0.5
  - α_false_negative * false_negative_rate * 0.3
  - α_success * success_rate * 0.2
```

其中 `α` 为各指标对应的自适应系数，可根据实际运营需求调整。

### 4. 策略衰退检测

系统持续监控策略有效性，自动检测性能下降趋势并触发重新演化。

**检测方法**：将当前评估窗口的数据分为前后两段进行对比

| 对比维度 | 检测指标 | 衰退判定 |
|----------|----------|----------|
| 处置准确率 | 前后半段准确率差值 | 下降 > 10% |
| 平均响应时间 | 前后半段响应时间对比 | 上升 > 20% |
| 用户反馈评分 | 前后半段评分均值对比 | 下降 > 15% |

**触发流程**：

1. 检测到策略有效性显著下降
2. 系统标记该策略为 `degrading` 状态
3. 生成衰退告警通知策略管理员
4. 自动触发重新演化流程
5. 演化完成后对比新旧策略效果

## 使用方式

### 查看策略列表

进入**策略管理页面**，可查看所有处置策略的概览信息：

| 字段 | 说明 |
|------|------|
| 策略名称 | 策略的唯一标识名称 |
| 置信度阈值 | 当前生效的置信度判定阈值 |
| 适应度评分 | 基于历史反馈的综合评分（0-100） |
| 评分等级 | `excellent` / `good` / `moderate` / `poor` |
| 趋势 | `持续上升` / `稳定` / `持续下降` |
| 状态 | `active` / `degrading` / `evolving` / `paused` |

### 触发演化

**手动触发**：

1. 在策略列表中选择目标策略
2. 点击「触发演化」按钮
3. 系统自动执行演化流程
4. 演化完成后展示新旧策略对比

**自动触发**：

当满足以下任一条件时，系统自动触发演化：

- 反馈数据积累量达到预设阈值（默认 1000 条）
- 策略衰退检测标记为 `degrading` 状态
- 适应度评分连续 2 次评估低于 `moderate` 等级

**演化流程**：

```
评估 → 检测衰退 → 生成建议 → 应用演化
```

1. **评估**：基于最新反馈数据重新计算策略适应度评分
2. **检测衰退**：对比历史数据，判断策略是否存在衰退
3. **生成建议**：Q-Learning 模型根据当前状态空间生成优化建议
4. **应用演化**：自动应用新策略参数，或提交人工审核后生效

### 查看演化历史

每次演化记录包含以下信息：

| 信息项 | 说明 |
|--------|------|
| 演化时间 | 演化的触发时间 |
| 旧规则 | 演化前的策略参数配置 |
| 新规则 | 演化后的策略参数配置 |
| 触发原因 | 手动触发 / 数据积累 / 衰退检测 / 评分过低 |
| 效果对比 | 演化前后的适应度评分和处置准确率对比 |
| 状态 | `succeeded` / `failed` / `rolled_back` |

## 最佳实践

1. **定期检查策略适应度评分**：建议每周至少检查一次策略评分，及时发现异常下降趋势
2. **关注策略衰退告警**：衰退告警是策略需要优化的早期信号，及时响应可避免处置质量持续下滑
3. **人工审核重大策略变更**：对于置信度阈值调整超过 10% 或涉及关键资产的策略变更，建议开启人工审核模式
4. **合理设置反馈采集周期**：反馈数据是演化的基础，确保处置结果反馈及时、准确录入系统
5. **监控探索与利用平衡**：关注探索率 ε 的衰减曲线，确保系统在稳定性和适应性之间保持平衡
6. **定期归档历史策略**：保留历史演化记录便于回溯分析，建议按季度归档一次
