# 多租户管理 (Multi-Tenant)

SecMind 的多租户管理功能为企业级用户提供完善的租户隔离与管理能力，满足 MSP/MSSP 服务商、大型企业多部门及集团子公司的安全运营需求。

## 功能概述

- **产品定位**：企业级租户隔离与管理
- **适用场景**：
  - MSP/MSSP 服务商：为多个客户提供独立的安全运营平台
  - 大型企业多部门：各部门独立运营，统一管理
  - 集团子公司：子公司间数据隔离，集团层面全局管控

## 核心特性

### 1. 数据隔离

- **PostgreSQL Row-Level Security (RLS)**：基于 PostgreSQL 行级安全策略实现数据隔离
- **自动过滤**：22 张业务表自动携带 `tenant_id` 过滤条件，确保数据隔离无死角
- **跨租户访问**：超级管理员可跨租户访问所有数据，便于全局监控与管理
- **租户内隔离**：普通用户仅可见本租户数据，无法感知其他租户存在

### 2. 租户生命周期

| 操作 | 说明 |
|------|------|
| 创建租户 | 自动分配套餐和配额，初始化租户配置 |
| 激活租户 | 启用已创建的租户，恢复正常服务 |
| 暂停租户 | 临时停止租户服务，保留数据和配置 |
| 删除租户 | 彻底删除租户及其关联数据（需谨慎操作） |
| 租户配置 | 自定义品牌 Logo、域名绑定、时区设置 |

### 3. 成员管理

- **邀请机制**：通过邮件邀请成员加入指定租户
- **角色分配**：

| 角色 | 权限范围 |
|------|----------|
| `admin` | 租户内最高权限，可管理成员和配置 |
| `soc_manager` | 安全运营管理权限，可编排处置流程 |
| `analyst` | 安全分析师权限，可查看和分析告警 |
| `viewer` | 只读权限，仅可查看仪表盘和报告 |

- **成员状态管理**：支持启用、禁用成员账号，保障人员变更时的安全性

### 4. 配额管理

**4 级套餐配额体系：**

| 套餐 | 用户数上限 | 告警/天 | 适用场景 |
|------|-----------|---------|----------|
| Free | 5 用户 | 100 条 | 个人或小团队试用 |
| Starter | 20 用户 | 1,000 条 | 小型企业入门 |
| Professional | 100 用户 | 10,000 条 | 中型企业标准运营 |
| Enterprise | 无限 | 无限 | 大型企业或 MSSP |

- **超限拦截**：达到配额上限时自动拦截新增操作，并发送告警通知
- **用量统计**：提供可视化的配额使用统计面板，支持按时间范围查看
- **套餐变更**：支持在线升级或降级套餐，变更即时生效

## 使用方式

### 超级管理员

超级管理员拥有系统最高权限，可执行以下操作：

1. **查看所有租户**：在管理后台查看全部租户列表及状态
2. **管理租户套餐**：为任意租户调整套餐、修改配额
3. **全局配置**：配置系统级别的全局策略和默认设置
4. **跨租户审计**：审计任意租户内的操作日志和数据

### 租户管理员

租户管理员（`admin` 角色）负责本租户的日常管理：

1. **管理本租户成员**：邀请新成员、分配角色、禁用或移除成员
2. **查看用量统计**：实时查看本租户的配额使用情况和趋势
3. **配置租户设置**：自定义品牌信息、绑定域名、设置时区
4. **操作审计**：查看本租户内的成员操作日志

### 普通用户

普通用户（`analyst` / `viewer` / `soc_manager`）仅可在本租户权限范围内工作：

1. **安全运营**：在分配的权限范围内处理告警和事件
2. **数据访问**：仅可见本租户的安全数据
3. **个人设置**：配置个人偏好，如通知方式和语言

## 数据隔离机制

SecMind 采用三层数据隔离架构，全方位保障租户数据安全：

### 第一层：中间件层

- 请求到达时，中间件自动从 JWT Token 中提取 `tenant_id`
- 将 `tenant_id` 注入请求上下文，供后续处理链路使用
- 非法或伪造的 `tenant_id` 请求将被拒绝

### 第二层：应用层

- 所有业务查询自动追加 `tenant_id` 过滤条件
- 通过统一的 ORM 封装层确保过滤逻辑的一致性
- 开发者无法绕过租户过滤机制直接访问数据

### 第三层：数据库层

- 基于 PostgreSQL RLS 策略的强制行级安全控制
- 数据库层面确保即使应用层出现漏洞，数据也不会泄露
- RLS 策略自动应用于所有关联的业务表

三层保障相互补充，有效防止数据泄露和越权访问。

## 最佳实践

### 域名规划

- 为每个租户分配独立域名，避免跨租户 Cookie 冲突
- 示例：`tenant-a.secmind.com`、`tenant-b.secmind.com`
- 使用反向代理统一处理多域名路由

### 成员管理

- 定期审查成员权限，及时清理不活跃账号
- 遵循最小权限原则，按需分配角色
- 人员离职时及时禁用或移除账号

### 配额监控

- 设置配额使用率告警阈值（建议 80% 触发预警）
- 定期查看用量统计趋势，提前规划套餐升级
- Enterprise 客户仍需关注存储和 API 调用等其他资源限制

### 安全审计

- 开启操作日志审计功能，记录关键操作
- 超级管理员定期抽查各租户运营情况
- 租户管理员关注本租户的异常登录和操作行为
