# 报表引擎

## 功能概述

报表引擎是 SecMind v3.0 的核心能力之一，定位为**多维度安全报告自动生成**平台。它连接数据采集、分析与呈现全链路，将分散的安全数据转化为结构化、可视化的专业报告。

**核心价值：从数据到洞察，一键生成专业报告。**

- **降本增效**：告别手工整理数据、制作图表、排版报告的繁琐流程，分钟级完成报告产出
- **标准化输出**：内置预设模板与合规框架，确保报告内容完整、格式统一、术语规范
- **灵活可扩展**：支持模板自定义、API 集成和多种导出格式，适配不同受众与交付场景
- **数据驱动决策**：通过趋势分析、分布统计、效率评估等维度，为安全运营提供数据支撑
- **定时自动推送**：支持定时生成并通过邮件、Webhook 等多种渠道自动推送报告
- **报告全生命周期管理**：提供报告生成、预览、存储、分享与归档的统一管理界面

## 预设模板

系统内置 6 种预设模板，覆盖日报、周报、月报及专项分析等典型场景。

### 模板清单

| 模板 | 说明 | SQL 查询 | 图表 | 适用场景 |
|------|------|----------|------|----------|
| 每日安全概览 | 今日告警/处置/指标汇总 | 7 | 3 | 日报 |
| 每周安全报告 | 周趋势/TOP威胁/MTTR | 8 | 4 | 周报 |
| 每月安全报告 | 月度汇总/趋势/建议 | 7 | 4 | 月报 |
| 攻击类型分布 | 攻击类型/严重程度分布 | 4 | 4 | 分析 |
| MTTR 分析 | 响应时间/效率分析 | 5 | 4 | 优化 |
| 处置效率 | 处置动作/成功率 | 6 | 4 | 评估 |

## 合规报表

报表引擎内置 4 种主流合规框架，可直接生成满足审计要求的合规报告。

| 框架 | 控制点 | 适用地区 |
|------|--------|----------|
| 等保 2.0 | 73 控制点 / 10 个域 | 中国 |
| GDPR | 31 条款 | 欧盟 |
| ISO 27001:2022 | 93 控制 / 4 主题 | 国际 |
| CIS Controls v8 | 18 控制 / 3 实施组 | 国际 |

合规报表的核心流程：

1. **选择框架** — 选择合规框架，系统自动关联对应的控制点映射表
2. **导入数据** — 导入合规检查结果数据（支持手动上传或对接合规管理模块）
3. **自动编排** — 报表引擎按框架结构自动编排章节，填充检查状态、证据列表、差距分析
4. **输出报告** — 输出标准合规报告，可直接用于内审或外部审计交付

## 图表类型

报表引擎支持以下 7 种可视化图表，可根据数据特征自动匹配或手动指定：

| 图表类型 | 标识 | 适用场景 |
|----------|------|----------|
| 柱状图 | `bar` | 对比不同分类的数值大小，如告警级别分布 |
| 饼图 | `pie` | 展示各部分占比，如攻击类型构成 |
| 折线图 | `line` | 反映趋势变化，如告警量日趋势 |
| 仪表盘 | `gauge` | 展示关键指标达成率，如 SLA 达标率 |
| 热力图 | `heatmap` | 呈现多维度密度分布，如告警时间-类型热力 |
| 雷达图 | `radar` | 多维度综合对比，如安全态势综合评分 |
| 树图 | `treemap` | 展示层级结构及占比，如资产风险分布 |

## 使用方式

### 生成报表

通过 Web UI 生成报表只需 4 步：

1. **选择模板** — 从预设模板或自定义模板中选择
2. **设置时间范围** — 指定报告覆盖的数据时间区间
3. **点击生成** — 系统自动执行 SQL 查询、渲染图表、编排报告
4. **预览 / 导出** — 在线预览确认无误后，选择格式导出

### 定时推送

支持配置定时任务，自动生成并推送报告到指定渠道：

```python
POST /api/v1/reports/schedules
{
  "name": "每周安全周报推送",
  "template_id": "weekly_security_report",
  "schedule": {
    "cron": "0 9 * * 1",
    "timezone": "Asia/Shanghai"
  },
  "delivery": [
    {
      "type": "email",
      "recipients": ["ciso@company.com", "soc-lead@company.com"]
    },
    {
      "type": "webhook",
      "url": "https://hooks.slack.com/services/xxx"
    }
  ],
  "format": "pdf"
}
```

### 自定义模板

当预设模板无法完全匹配业务场景时，可基于模板引擎创建自定义模板，自由组合 SQL 查询、图表和文本段落：

```python
POST /api/v1/reports/templates
{
  "name": "月度安全简报",
  "description": "面向管理层的月度安全运营简报",
  "sections": [
    {
      "id": "summary",
      "title": "安全态势概览",
      "type": "markdown",
      "content": "本月共处理告警 {{alert_count}} 条，其中高危 {{high_count}} 条..."
    },
    {
      "id": "alert_chart",
      "title": "告警趋势",
      "type": "chart",
      "chart_type": "line",
      "data_source": "alerts",
      "group_by": "day",
      "metrics": ["count", "high_count", "critical_count"]
    },
    {
      "id": "top_threats",
      "title": "Top 威胁",
      "type": "table",
      "data_source": "threats",
      "columns": ["threat_type", "count", "severity", "status"],
      "sort_by": "count",
      "limit": 10
    }
  ]
}
```

## 导出格式

支持多种导出格式，满足不同交付场景需求：

| 格式 | 说明 | 适用场景 | 状态 |
|------|------|---------|------|
| `pdf` | 格式化报告，包含排版、图表、封面等 | 正式报告、审计交付 | 规划中 |
| `docx` | Word 文档 | 需要编辑的报告 | 已支持 |
| `html` | HTML 页面 | 在线查看 | 已支持 |
| `xlsx` | Excel 表格 | 数据分析 | 已支持 |
| `json` | 完整数据结构 | 二次处理与 API 对接 | 已支持 |
| `csv` | 表格数据 | Excel 打开或导入其他系统 | 已支持 |

## API 调用

报表引擎提供以下 REST API（完整文档请参考[报告生成 API](/api/reports)）：

```bash
# 获取模板列表
GET /api/v1/reports/templates

# 创建自定义模板
POST /api/v1/reports/templates

# 生成报表
POST /api/v1/reports/generate
{
  "template_id": "daily_security_overview",
  "time_range": {
    "start": "2026-05-15T00:00:00Z",
    "end": "2026-05-15T23:59:59Z"
  }
}

# 导出报表
POST /api/v1/reports/export
{
  "report_id": "rpt_20260516_001",
  "format": "json"
}

# 配置定时推送
POST /api/v1/reports/schedules
```

### 配置选项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `default_format` | 默认导出格式 | `pdf` |
| `max_report_size_mb` | 最大报告大小 | `50` |
| `chart_dpi` | 图表分辨率 | `150` |
| `include_raw_data` | 是否包含原始数据附件 | `false` |
| `watermark` | 报告水印 | `CONFIDENTIAL` |

## 最佳实践

- **设置定时报表自动生成**：针对日报、周报等周期性场景，配置定时任务让系统自动生成并推送，减少人工干预
- **使用合规报表满足审计要求**：在合规检查完成后，直接通过报表引擎生成对应框架的合规报告，确保控制点全覆盖、证据链完整
- **自定义模板满足特定需求**：当预设模板无法完全匹配业务场景时，基于模板引擎创建自定义模板，自由组合 SQL 查询、图表和文本段落
- **模板标准化**：建立标准化的报告模板体系，确保跨团队、跨周期的报告格式一致、口径统一
- **面向受众**：根据报告受众（管理层、技术团队、审计方）调整内容深度和呈现方式，提升报告可读性
- **数据驱动**：报告内容应以数据为支撑，避免主观描述，确保每项结论都有量化依据
